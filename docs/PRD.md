## 1. Описание продукта

### Проблема
Студенты теряют время в очередях в университетской столовой в обеденный перерыв. Сотрудники столовой не знают заранее сколько блюд готовить, что приводит к нехватке еды или большим остаткам.

### Решение
Веб-приложение для **предзаказа блюд онлайн** с выбором временного слота получения. Студент заказывает заранее — приходит в нужное время и сразу забирает готовый заказ без ожидания.

### Целевая аудитория
- **Студенты** — основные пользователи, делают заказы
- **Сотрудники столовой** — принимают и обрабатывают заказы
- **Администраторы** — управляют меню, пользователями, статистикой

## 3. Нефункциональные требования

| ID | Требование | Значение |
|----|-----------|----------|
| NF-01 | Время ответа API | ≤ 500 мс для 95% запросов |
| NF-02 | Безопасность | JWT-токены, bcrypt хеширование паролей |
| NF-03 | Покрытие тестами | ≥ 60% (pytest + Vitest) |
| NF-04 | Доступность | Работает в современных браузерах (Chrome, Firefox, Edge) |
| NF-05 | Код | Статический анализ: ruff (Python), ESLint (TypeScript) |
| NF-06 | CI/CD | Автоматические тесты при каждом push (GitHub Actions) |
| NF-07 | БД | PostgreSQL, данные не теряются между перезапусками |
| NF-08 | Масштабируемость | Архитектура позволяет добавлять новые модули |

## 4. Технический стек

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| Backend | Python 3.12, FastAPI | Быстрая разработка REST API, автодокументация (Swagger) |
| ORM | SQLAlchemy 2.0 | Мощный ORM с поддержкой PostgreSQL |
| База данных | PostgreSQL 16 | Надёжная реляционная СУБД |

| Frontend | React 18, TypeScript | Компонентный подход, типобезопасность |
| State management | Zustand | Простой, без boilerplate |
| HTTP клиент | Axios | Перехватчики запросов, интерцепторы |
| Линтеры | ruff, ESLint | Статический анализ кода |
| CI/CD | GitHub Actions | Бесплатно, интегрировано с GitHub |
| Тесты (backend) | pytest, pytest-cov | Стандарт для Python |
| Тесты (frontend) | Vitest, Testing Library | Совместим с Vite |

## 5. Паттерны проектирования

| Паттерн | Где применяется | Зачем |
|---------|----------------|-------|
| **Repository Pattern** | `app/repositories/` | Изоляция логики доступа к данным от бизнес-логики |
| **Service Layer** | `app/services/` | Бизнес-логика вынесена из роутеров |
| **Factory** | `OrderService.create_order()` | Создание сложного объекта Order с валидацией |
| **Dependency Injection** | FastAPI `Depends()` | Внедрение зависимостей (БД, пользователь) |
